# Challenge SC01E04 - Documentation & TypeScript

## Rappel : Git Flow

Penser à mettre à jour votre dépôt à l'aide du Git Flow, et à créer une branche dédiée pour réaliser le challenge.

## Objectif 

Aujourd'hui, pas vraiment de challenge : on s'applique à **lancer l'API en local et à la documenter.**

## Lancement de l'API 

Après avoir fait le Git Flow : 
- installer les dépendances du projet
- créer une BDD Postgres dédiée au projet (`oquiz/oquiz/oquiz`)
  - attention, il faudra le droit `CREATEDB` sur l'utilisateur pour Prisma
- créer un fichier `api/.env` pour gérer les variables d'environnement
- lancer les migrations
- lancer l'échantillonnage (seeding)
- lancer le serveur API
- le tester (Chrome, RestClient, ThunderClient...)

## Documentation

Créer et compléter un fichier `INSTALL.md` pour documenter l'installation et le lancement du projet en local.
