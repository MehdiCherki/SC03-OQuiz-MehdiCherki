# Web Design - UI/UX

- **UI** = `User Interface`
- **UX** = `User eXperience`


## 📊 Résumé

| Étape         | Objectif                    | Niveau de détail | Visuel  | Interactif ? |
| ------------- | --------------------------- | ---------------- | ------- | ------------ |
| **Zoning**    | Répartir l’espace           | Faible           | Blocs   | ❌ Non       |
| **Wireframe** | Structurer sans design      | Moyen            | Schéma  | ❌ Non       |
| **Mockup**    | Montrer le design final     | Élevé            | Complet | ❌ Non       |
| **Prototype** | Simuler l’expérience réelle | Élevé            | Complet | ✅ Oui       |


## 🟫 Zoning  

- **But** : Définir les **grandes zones fonctionnelles** d’une interface (entête, menu, contenu, pied de page…).  
- **Aspect** : Très basique, souvent en gris ou avec des blocs étiquetés.  
- **Détail** : Très faible (ni texte précis, ni design).  
- **Quand** : Première étape de réflexion.
- **Outil** : Papier, Balsamiq, **TLDraw**, Excalidraw, Paint


## 📐 Wireframe  (maquette filaire)
- **But** : Définir la **structure détaillée** d’une page, sans design graphique. Expliciter les interactions entre l'utilisateur et la page. Outils de conception, pour aider à comprendre le besoin.
- **Aspect** : Schéma en noir et blanc avec zones précises (textes, boutons, formulaires).
- **Détail** : Moyen à élevé, mais sans style (polices, couleurs, images réelles).  
- **Notes** : Plusieurs variations d'une même page (ex : mobile, desktop) (ex : admin, membre)
- **Quand** : Après le zoning, pour structurer précisément les écrans.
- **Outil** : Papier, Wireframe.cc, **Whimsical**, Balsamiq, TLDraw, Excalidraw, **Figma**.


## 🎨 Mockup (maquette design)
- **But** : Montrer le **rendu visuel final** (polices, couleurs, images, branding). 
- **Aspect** : Presque identique à un vrai site/app, mais **non interactif**.  
- **Détail** : Très élevé (look & feel complet).
- **Quand** : Après validation des wireframes, pour validation graphique.
- **Outil** : **Figma**, Sketch, Pen Pot, Adobe XD.


## 🧪 Prototype  
- **But** : Simuler **l’expérience utilisateur complète**, avec interactions.  
- **Aspect** : Comme un mockup, mais **cliquable** et **navigable**.  
- **Détail** : Très élevé + interactions (liens, animations, transitions).  
- **Quand** : Avant développement, pour tests utilisateurs.
- **Outil** : **Figma** (prototypage), Adobe XD, InVision.
